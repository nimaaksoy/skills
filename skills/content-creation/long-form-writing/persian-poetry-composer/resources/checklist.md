# Pre-delivery checklist

Run before sending the poem back to the user. Every poem fails the first time on at least one of these.

---

## Brief

- [ ] Subject from the user's words is reflected in the poem (not just abstractly).
- [ ] Style requested (or proposed and accepted) is the style actually delivered.
- [ ] Length matches the request (or matches the form's default if no length was given).

## Vazn (meter)

- [ ] Vazn is named in the **Style notes** block.
- [ ] Every مصراع scans against the chosen vazn — no missing or extra syllables.
- [ ] No mid-poem switch in vazn.
- [ ] Read aloud: the rhythm is felt, not just diagrammed.

## Qafiye (rhyme) and Radif

- [ ] قافیه pattern matches the form (غزل: aa, ba, ca…; رباعی: a, a, b, a; مثنوی: aa, bb, cc…).
- [ ] Rhymes are aural, not just visual.
- [ ] If radif is used, it appears in **every** rhyming مصراع, identical each time.
- [ ] No identical-word "rhyme" (repetition isn't rhyme).

## Style discipline

- [ ] **Saadi**: at least one syntactic surprise; no archaic vocabulary for its own sake.
- [ ] **Hafez**: each beyt is anchored by a دو-پهلو word; you can name the two specific readings on demand.
- [ ] **Rumi**: at least one imperative or call; repetition shifts intensity each time, not just repeats.
- [ ] **Ferdowsi**: every beyt advances narrative, sharpens scene, or names a virtue — none decorate idly.
- [ ] **Khayyam**: مصراع 4 lands as both surprising and inevitable; cover it and verify a reader couldn't predict it from 1–3.
- [ ] **Modern lyric**: line length is consistent within each section; chorus has a sustained-vowel hook word.

## Persian quality

- [ ] Persian script is used in the main poem (not romanisation).
- [ ] No ezafe errors that break scansion.
- [ ] No consonant clusters at line ends that make it unsingable / unreadable aloud.
- [ ] No Arabic-heavy phrasing in a piece that asked for spoken Persian.

## Output blocks

- [ ] **شعر** block delivered in Persian script.
- [ ] **توضیح سبکی** block names the style, vazn, qafiye/radif, and the signature move.
- [ ] **ترانویسی** included only if user asked or appears non-Persian-reading.
- [ ] **یادداشت آهنگ‌گذاری** included if the poem is song-shaped, otherwise skipped.

## Honesty

- [ ] If you couldn't make the chosen vazn work cleanly, you said so and offered a free-verse version instead — you didn't ship a broken classical meter and call it Hafez.
- [ ] If the رباعی's fourth مصراع is weak, you didn't ship it. You either rewrote 1–3 or proposed a different form.
- [ ] If the user's brief was genuinely ambiguous, you asked one question instead of guessing three.

If any box is unchecked, fix it before delivering. The user can tell.
